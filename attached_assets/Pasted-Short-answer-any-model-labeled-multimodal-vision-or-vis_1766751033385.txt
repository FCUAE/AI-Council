Short answer: any model labeled “multimodal”, “vision”, or “vision-capable” supports image input. Which exact model names depend on the provider and on release timing. Below are common examples, how to check, and a quick test snippet you can run.

Examples of vision-capable models (representative, not exhaustive)

OpenAI family
GPT-4V / “GPT‑4 Vision” and vision-enabled variants (e.g., gpt-4v, gpt-4o-mini-vision in some provider catalogs).
Google
Gemini multimodal variants (Gemini Pro/Ultra or other “multimodal” Gemini models).
Anthropic
Claude multimodal variants (Claude 3+ family has multimodal offerings in many releases).
Open-source / research VLMs
LLaVA, BLIP-2, MiniGPT-4, OpenFlamingo, Vision-LLaMA and other vision-language models — these accept images and produce text.
Other vendors
Many vendors (Mistral, Cohere, etc.) provide multimodal/vision-enabled models; check vendor docs for a “vision” or “multimodal” tag.
How to confirm if a model supports images

Check the model documentation or the model metadata (often a capabilities or modalities field listing image).
Look for words like “vision”, “multimodal”, or “image understanding” in the model description.
Try a tiny test request (see example below). If the model accepts a file or an image MIME (or base64 image) and returns analysis, it’s vision-capable.
Generic test example (multipart upload)

Replace https://api.example.com/v1/predict, MODEL_NAME, and API_KEY with your provider’s values.

curl -X POST "https://api.example.com/v1/predict" \
  -H "Authorization: Bearer API_KEY" \
  -F "model=MODEL_NAME" \
  -F "prompt=Describe and analyze this image" \
  -F "image=@/path/to/photo.jpg"
Generic test example (base64 JSON)

Some APIs accept base64-encoded images in JSON.

curl -X POST "https://api.example.com/v1/predict" \
  -H "Authorization: Bearer API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "MODEL_NAME",
    "prompt": "Describe this image",
    "image_base64": "data:image/jpeg;base64,/9j/4AAQ..."
  }'
Notes and caveats

Availability, pricing, and exact model names change often — always verify the provider’s docs.
Some models can accept images but have limits (max size, number of images, types of outputs like bounding boxes vs. plain text).
Vision-capable models still follow content policy and may have restricted capabilities for sensitive images (faces, medical, etc.).
